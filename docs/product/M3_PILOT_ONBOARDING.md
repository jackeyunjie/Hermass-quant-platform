# M3 Controlled Pilot Onboarding 流程与免责声明

## 适用范围

本文档定义 Hermass M3 阶段（2026-06-23 起）的受控试点用户 onboarding 流程。

- **试点人数**：5 人（首轮），累计不超过 20 人
- **试点周期**：2 周观察期
- **使用范围**：策略研究、回测验证、假设检验
- **禁止范围**：投资建议、向他人推荐、真实交易、资金托管

## 用户画像（5 人目标）

| # | 画像 | 预期层级 | 价值 |
|---|---|---|---|
| 1 | 有 3-5 年主观交易经验，想系统化表达策略 | H1 → H2 | 验证 DSL 生成是否 intuitive |
| 2 | 用过聚宽/米筐，有量化基础但策略不稳定 | H2 → H3 | 验证红线诊断是否帮助发现盲区 |
| 3 | 数据分析师，对金融感兴趣但无交易经验 | H1 | 验证新手引导是否足够 |
| 4 | 私募研究员，需要快速验证因子假设 | H3 | 验证 evidence 深度是否满足专业需求 |
| 5 | 技术开发者，想理解 Hermass 架构 | H2/H3 | 验证 API 和文档是否完整 |

## Onboarding 流程

### Step 1: 邀请发送（人工）

发送渠道：私信/邮件/微信（不公开链接）

内容模板：

```
Subject: Hermass 策略研究工具 - 邀请制试点

Hi [Name],

你被选为 Hermass 首批试点用户。Hermass 是一个 AI 原生的量化策略研究平台，
当前处于非常早期的研究工具阶段，不是投资建议服务。

试点目标：帮助我们验证"中文策略想法 → 结构化 DSL → 回测验证 → 交易证据"
这条链路是否对研究者有价值。

开始前，请阅读并确认免责声明：[链接到 /onboarding/disclaimer]

预计时间投入：首次 30 分钟体验 + 第 7 天和第 14 天各 5 分钟反馈问卷。

[邀请链接 - 含唯一 token，7 天有效]
```

### Step 2: 免责声明签收（强制）

用户必须逐项勾选以下声明，全部勾选后方可进入系统：

---

**Hermass 免责声明**

1. [ ] 我理解 Hermass 是策略研究工具，不是投资建议服务。
2. [ ] 我不会把 Hermass 的输出当作买卖指令或向他人推荐。
3. [ ] 我理解回测结果不代表未来收益，历史表现不能预测未来。
4. [ ] 我接受所有策略必须经过 DSL 结构化、红线检查和回测验证。
5. [ ] 我不会要求跳过止损、仓位限制或审计等安全约束。
6. [ ] 我理解当前数据可能存在停牌、退市、复权等处理边界，结果仅供研究参考。
7. [ ] 我愿意把使用过程中的问题和建议反馈给 Hermass 团队。

[确认并进入] [取消]

---

签收记录：
- 存储到 `audit_db.onboarding_consent`（含 timestamp、IP、勾选项）
- 与后续所有操作共享同一 `trace_id` 前缀

### Step 3: 分层诊断（3-5 题）

见 `docs/product/H1_H2_H3_USER_DIAGNOSIS.md`

诊断结果：
- 推荐 H1/H2/H3 入口
- 用户可手动切换
- 记录到 `audit_db.onboarding_diagnosis`

### Step 4: 首次体验引导（按层级）

见 `docs/product/H1_H2_H3_USER_DIAGNOSIS.md` 首次体验引导章节

### Step 5: 反馈收集

**第 7 天问卷**（自动发送，邮件/站内）：

1. 你当前主要使用的功能是？（H1/H2/H3）
2. 你成功创建/验证了几个策略？
3. 遇到过哪些阻塞或困惑？
4. 红线检查对你有帮助吗？（1-5 分）
5. 回测结果的可解释性如何？（1-5 分）
6. 你会向同事推荐 Hermass 吗？（0-10 分，NPS）

**第 14 天问卷**：

1. 过去 7 天你使用了几次 Hermass？
2. 有没有基于 Hermass 的输出修改过你的策略想法？
3. 你最希望增加的功能是什么？
4. 你最希望改进的体验是什么？
5. 如果 Hermass 正式开放，你会付费使用吗？（是/否/看价格）

## 停止标准

出现以下任一情况，立即暂停试点并评估：

| 触发条件 | 响应 |
|---|---|
| 用户要求跳过红线或认为红线是"平台不够聪明" | 1:1 沟通，若坚持则移除试点资格 |
| 用户将回测结果作为投资建议传播（截图/转发） | 立即暂停，法律 review |
| 系统性能导致单次操作 >60s 等待 | 技术排查，优化前暂停新用户进入 |
| 数据质量投诉（停牌/退市/复权问题） | 记录具体问题，评估是否修复后再开放 |
| 2 人及以上在 NPS 中评分 <5 | 产品 review，决定是否调整方向 |

## 数据与隐私

- 用户策略 DSL、回测结果、audit 记录仅用于产品改进，不用于商业目的
- 不收集用户真实交易数据、资金信息、身份敏感信息
- 试点结束后，用户可选择删除所有个人数据
- 匿名化统计数据可用于产品演示（需用户额外授权）

## 试点成功标准

2 周后评估是否进入 M4（Public Beta）或延长试点：

| 指标 | 目标 | 测量方式 |
|---|---|---|
| 5 人完成首次体验 | 100% | onboarding completion rate |
| 平均创建策略数 | >=3 | audit_db 统计 |
| 红线理解率 | >=80% | 问卷 Q4 |
| NPS | >=30 | 问卷 Q6 |
| 无合规事件 | 0 | 人工监控 |
| 性能满意度 | >=3/5 | 问卷 |
