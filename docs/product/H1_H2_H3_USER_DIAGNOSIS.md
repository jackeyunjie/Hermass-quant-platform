# H1 / H2 / H3 用户进入诊断表

## 用途

用户在首次进入 Hermass 时，通过 3-5 个问题自动匹配到最适合的研究场域（H1/H2/H3）。

不是"能力测试"，而是"需求匹配"——帮助用户找到当前最需要的入口，避免 H3 复杂功能吓到 H1 新手，也避免 H1 基础功能让 H3 用户觉得浪费时间。

## 诊断问题集

### 问题 1：你当前的状态是？

| 选项 | 分值 | 说明 |
|---|---|---|
| 我有一个策略想法，但不知道怎么表达清楚 | H1 +3 | 需要 DSL 结构化帮助 |
| 我已经有明确的策略规则（比如 MA 交叉 + 止损） | H2 +2 | 需要验证和诊断 |
| 我想用真实数据验证我的策略假设，并复盘每笔交易 | H3 +3 | 需要证据和回测 |
| 我想找一个能自动赚钱的系统 | ❌ 拒绝 | 不适合 Hermass |

### 问题 2：你对策略回测的理解是？

| 选项 | 分值 | 说明 |
|---|---|---|
| 回测就是看看历史收益率 | H1 +1 | 基础认知，需要教育 |
| 回测需要看每笔交易的进出理由和当时的市场状态 | H3 +2 | 理解证据驱动 |
| 回测结果不代表未来，但可以帮助我理解策略在什么环境下有效 | H3 +3 | 理解假设-证据循环 |
| 我不关心回测，只关心明天买什么 | ❌ 拒绝 | 不适合 Hermass |

### 问题 3：你能接受以下哪种结果？

| 选项 | 分值 | 说明 |
|---|---|---|
| 系统告诉我策略哪里不严谨，帮我修正 | H2 +2 | 接受诊断和红线 |
| 系统给我完整的交易记录，让我自己判断策略好坏 | H3 +2 | 接受证据自主判断 |
| 系统直接告诉我策略好不好，能不能赚钱 | ❌ 拒绝 | 不适合 Hermass |
| 我还没想好，想先试试 | H1 +1 | 允许探索 |

### 问题 4（可选）：你有过量化交易经验吗？

| 选项 | 分值 | 说明 |
|---|---|---|
| 完全没有 | H1 +1 | 从零开始 |
| 用过 Excel/同花顺等工具做过简单筛选 | H2 +1 | 有基础工具经验 |
| 写过策略代码或用过量化平台（聚宽、米筐等） | H3 +1 | 有技术基础 |
| 管理过实盘量化策略 | H3 +2 | 专业用户 |

## 路由规则

```
总分计算：
- H1 得分 = 问题1选A + 问题2选A + 问题3选D + 问题4选A/B
- H2 得分 = 问题1选B + 问题2选B + 问题3选A + 问题4选B/C
- H3 得分 = 问题1选C + 问题2选B/C + 问题3选B + 问题4选C/D

路由逻辑：
1. 任一问题选"拒绝" → 显示免责声明，建议不适合当前使用
2. H3 >= 5 且 H3 > H2 且 H3 > H1 → 推荐 H3（Evidence Lab）
3. H2 >= 4 且 H2 > H1 → 推荐 H2（Diagnosis）
4. 其他 → 推荐 H1（Structuring）

用户可手动切换：推荐不是强制，用户可选择"我想从基础开始"或"我想直接进高级"。
```

## 首次体验引导

### H1 引导路径

1. 展示 3 个冻结样例的 DSL 结构（MA5/MA20 策略）
2. 让用户用自然语言描述一个简单策略
3. 展示 DSL 生成结果，解释每个字段含义
4. 展示红线检查（如果缺少止损会被拒绝）
5. 完成一次 mock preview，看到条件命中数量

### H2 引导路径

1. 展示一个已有 DSL 的预览结果
2. 展示红线检查通过/失败的不同结果
3. 展示 preview 中"fully supported" vs "requires backtest context"的区别
4. 让用户修改一个参数，观察 preview 变化

### H3 引导路径

1. 展示一个完整策略的 backtest 结果（非 stub）
2. 展示 trade records 和 event evidence
3. 展示"为什么进、为什么出、当时状态是什么"
4. 展示 audit timeline，解释 trace_id 追溯
5. 让用户修改一个 exit 条件，重新跑回测，对比结果
