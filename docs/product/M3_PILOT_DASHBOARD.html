试点监控教案

每日 Audit 检查

Codex 负责每日检查以下数据表：

-- 新用户同意免责声明
SELECT COUNT(*) FROM onboarding_consent 
WHERE date(created_at) = CURRENT_DATE;

-- 新用户完成诊断
SELECT COUNT(*) FROM onboarding_diagnosis 
WHERE date(created_at) = CURRENT_DATE;

-- 策略生成次数
SELECT COUNT(*) FROM strategy_audit_log 
WHERE operation='generation' 
AND date(created_at) = CURRENT_DATE;

-- 反馈问卷提交
SELECT COUNT(*) FROM onboarding_feedback 
WHERE day IN (7, 14);

关键指标追踪

试点期间必须追踪的 6 项指标：

Onboarding 完成率 目标: 100%
平均创建策略数 目标: ≥3
红线理解率 目标: ≥80%
NPS 评分 目标: ≥30
平均回测耗时 目标: <20s
合规事件 目标: 0

停止标准

出现以下任一情况，立即暂停试点：

用户要求跳过红线检查
用户传播回测结果作为投资建议
单次操作等待 >60s
数据质量投诉（停牌/退市/复权）
2 人及以上 NPS <5
