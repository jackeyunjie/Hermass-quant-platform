# External Service Readiness And Pilot Plan

## Service Readiness Verdict

**CONTROLLED_PILOT_ONLY**

当前 Hermass AI Quant Platform 的 MVP 链路（中文策略输入 -> DSL v2 -> Schema/Pydantic 校验 -> 红线检查 -> 条件命中预览 -> Light Backtest -> 审计落库）已在样例级通过验收（5/5 cases passed，197 tests passed），但以下关键条件不满足 public beta 要求：

1. Light Backtest 仍为 `light_stub` 模式，未接入真实历史数据跑通完整权益曲线。
2. 对外服务文案、免责声明、用户协议尚未经过合规/法律复核。
3. 未在真实用户场景中验证端到端闭环。
4. 无生产级运维、监控、故障处理机制。

因此，当前阶段只允许**受控邀请制研究试点**，禁止公开商业化开放。

## Prohibited Services

| 禁止服务 | 原因 | 越界风险 |
|----------|------|----------|
| 投资建议（买/卖/持有具体标的） | 可能触发投资顾问监管 | 用户将系统输出理解为交易指令 |
| 荐股/买卖点承诺 | 直接违反证券法规 | 系统暗示某时某价应交易 |
| 保证收益或保本承诺 | 欺诈风险 | 用户基于虚假承诺决策 |
| 真实下单/交易执行 | 无券商接口、无风控、无合规 | 资金损失、法律追责 |
| 资金托管 | 无牌照、无监管 | 非法集资风险 |
| 用户 Python 策略代码执行 | 安全风险、不可控 | 代码注入、数据泄露 |
| LLM 生成代码执行 | 不可预测、不可审计 | 隐藏交易逻辑、后门 |
| 将 `light_stub` 包装为真实绩效 | 误导用户 | 用户基于假回测做真实投资 |
| 作为自动荐股/自动赚钱产品宣传 | 虚假宣传 | 监管处罚、用户诉讼 |
| 面向不特定公众的公开注册 | 无合规准备 | 无法管控用户预期和风险 |

4. **数据授权**：如使用用户提供的自定义数据，需确认数据来源合法。

## Pilot Service Flow

### Step 1: 用户提交中文策略想法

- **用户看到**：简洁输入框，提示"用中文描述您的策略想法，例如：MA5上穿MA20买入，跌破MA10卖出，止损8%"
- **系统内部**：接收字符串，准备进入解析
- **投资建议风险**：低 — 仅收集用户想法
- **如何避免越界**：输入框下方常驻免责声明
- **是否审计**：是 — 记录 `generation` operation

### Step 2: 系统生成 DSL 和人类解释

- **用户看到**：结构化策略卡片（买入条件、卖出条件、风控参数）+ 人类可读解释
- **系统内部**：`NLToDSLParser.parse()` 规则解析，生成 `StrategyDSL`
- **投资建议风险**：中 — 用户可能将结构化输出理解为"系统认可该策略"
- **如何避免越界**：
  - 明确标注"这是您想法的结构化表达，不代表平台推荐"
  - 展示 `SEMANTIC_WRONG_SECTION` 等 warning（如有）
- **是否审计**：是 — 记录 `generation` operation

### Step 3: 系统执行红线检查

- **用户看到**：校验结果（通过/失败）、触发的红线规则
- **系统内部**：`validate_dsl()` 执行结构校验 + 红线检查
- **投资建议风险**：低 — 平台内部约束检查
- **如何避免越界**：文案明确"红线检查是平台安全约束，不评价策略优劣"
- **是否审计**：是 — 记录 `validation` operation 含 `red_line_result`

### Step 4: 系统展示 Preview

- **用户看到**：各条件在历史数据中的估算命中数、支持状态（fully_supported / requires_backtest_context）
- **系统内部**：`PreviewService.preview()` 基于 mock 或 DuckDB 返回估算
- **投资建议风险**：中 — 用户可能将命中数理解为"该策略有效"
- **如何避免越界**：
  - 明确标注"估算基于历史数据，不代表未来表现"
  - `requires_backtest_context` 条件明确提示"需完整回测验证"
- **是否审计**：是 — 记录 `preview` operation

### Step 5: 系统执行 Light Backtest

- **用户看到**：回测指标卡片（total_return、sharpe_ratio、max_drawdown 等）
- **系统内部**：`run_dsl_backtest()` stub 返回占位指标
- **投资建议风险**：**高** — 用户极易将回测指标理解为"策略历史表现"
- **如何避免越界**：
  - **必须醒目展示**："当前回测为演示框架，指标值为占位符，非真实绩效"
  - 指标卡片顶部常驻 `mode="light_stub"` 标签
  - 每个指标旁标注"演示值"
- **是否审计**：是 — 记录 `backtest` operation，metrics 含 `"_mode": "light_stub"`

### Step 6: 系统输出报告

- **用户看到**：策略摘要 + 校验结果 + 预览估算 + 回测框架 + 免责声明页脚
- **系统内部**：组装 `MvpE2EResult` 为报告
- **投资建议风险**：中 — 报告整体可能被截图传播
- **如何避免越界**：
  - 每页页脚包含"研究用途，不构成投资建议"
  - 报告水印或页眉标注"HERMASS RESEARCH PILOT — NOT FOR TRADING"
- **是否审计**：报告生成本身不单独审计，但各阶段已审计

### Step 7: 系统写入 Audit

- **用户看到**：可选"查看审计历史"入口
- **系统内部**：写入 `strategy_audit_log` 和 `strategy_backtests`
- **投资建议风险**：无
- **是否审计**：本身就是审计

### Step 8: 用户决定是否修改策略

- **用户看到**："修改策略"按钮或重新输入
- **系统内部**：循环回到 Step 1，生成新版本
- **投资建议风险**：低
- **如何避免越界**：每次重新生成均重新展示免责声明

## Engineering Go/No-Go Checklist

### Go 条件（必须全部满足）

- [x] MVP acceptance command 通过：`scripts/run_strategy_lab_mvp_e2e_acceptance.py` 5/5 passed
- [x] 所有输出标注 `mode="light_stub"` 或 `"_mode": "light_stub"`
- [x] `light_stub` 结果在报告中明显标注为"演示框架"
- [x] 每次 generation/validation/preview/backtest 都有 `trace_id` 和 audit 记录
- [x] 禁止用户上传或执行 Python 代码（当前无此入口）
- [x] 用户输入、DSL、报告、审计记录可追踪（`input_hash`/`output_hash`）
- [x] 错误信息不泄露内部路径、token 或数据库结构
- [x] 红线检查强制阻止无止损、超仓位的策略进入 preview/backtest
- [x] 失败链路（如红线检查失败）也写入 audit（generation + validation）

### No-Go 条件（任一触发则暂停试点）

- [ ] `light_stub` 指标未标注或可被误解为真实绩效
- [ ] 无审计记录或审计记录不完整
- [ ] 存在用户代码执行入口
- [ ] 错误信息泄露敏感信息
- [ ] 免责声明未展示或被用户可跳过
- [ ] 服务文案出现禁止话术
- [ ] 未通过最近一次 MVP acceptance（当前 197 passed）

## Upgrade Criteria

从邀请制试点（Controlled Pilot）升级到公开 Beta（Public Beta）必须满足：

| 条件 | 当前状态 | 达标标准 |
|------|----------|----------|
| 真实 Light Backtest | stub | DuckDB + Polars 真实历史数据回测，P50 < 30s |
| 合规复核通过 | 未启动 | 律师确认服务不构成投资顾问；免责声明有效 |
| 试点用户闭环 | 0 | 至少 10 个试点用户完成 >=3 次策略迭代 |
| 审计记录完整 | 样例级 | 所有试点用户的全部操作可审计、可查询 |
| 故障/误解/投诉处理 | 未建立 | 有明确处理流程和责任人 |
| GitHub/Obsidian 同步 | K10 已建立 | 每次迭代后同步纪律执行 |
| 服务文案无禁止话术 | 已建立清单 | 律师复核通过 |
| 用户协议/隐私政策 | 未起草 | 律师起草、用户可签署 |
| 数据安全/备份 | 本地 DuckDB | 生产级存储 + 备份策略 |
| 监控/告警 | 无 | 核心链路异常可告警 |

**结论**：当前距离 public beta 至少还需要完成真实回测引擎、合规复核、用户协议、试点闭环验证四项核心工作。

| 功能 | 开放 | 说明 |

| 标准 | 指标 |
|------|------|
